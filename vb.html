Chapter 01
Introduction to Visual Basic
What is Visual Basic?
▼
Visual Basic (VB6) is a programming language developed by Microsoft that allows you to build Windows applications using a simple drag-and-drop interface combined with code.
💡
Think of it like this: VB6 is like building with LEGO blocks. You drag ready-made pieces (controls like buttons and textboxes) onto a form, then write instructions for what happens when a user clicks or types.
GUI
Graphical User Interface – windows, buttons, menus
Event-Driven
Code runs when user actions happen (click, type, etc.)
IDE
Integrated Development Environment – your coding workspace
🧠 Quick Check: VB6 programming style is called...
Procedure-Driven
Event-Driven
Object-Oriented
Event-Driven means code runs in response to user actions like button clicks and key presses.
VB Editions & Philosophy
▼
VB6 came in 3 editions to suit different needs.
L
Learning Edition – For beginners learning the language. Basic controls included.
P
Professional Edition – Full features including ActiveX, internet controls and database tools.
E
Enterprise Edition – For teams building large-scale client/server applications.
📌
VB Philosophy: "Make programming easy." VB6 hides complex Windows programming behind a visual, beginner-friendly environment.
Controls, Properties, Events & Methods
▼
🏠
Real-life analogy: Think of a Button as a light switch.
• Control = the switch itself
• Property = its color, label ("ON/OFF")
• Event = when you flip it
• Method = the action that follows (light goes on)
Control
A visual object: Button, TextBox, Label, etc.
Property
Attribute of a control: Name, Caption, Color, Size
Event
User action: Click, KeyPress, MouseMove, Load
Method
Action a control can do: Move, Show, Hide, Print
🧠 Which of these is a Property of a Button?
Click
Caption
Show
Caption is a property — it's the text shown on the button. Click is an Event. Show is a Method.

Creating an Application
The VB6 IDE Windows
▼
When you open VB6, you see several windows — each has a specific purpose.
1
Toolbox – A panel with all controls (Button, Label, TextBox, etc.) you can drag onto your form.

Form Events – The Lifecycle
▼
🔄
Form Lifecycle: Forms follow a sequence of events from birth to death. Understanding this order is crucial!
1
Initialize – Form object is created. Set default values here.
2
Load – Form loads into memory. Load data, set up controls here.
3
Activate – Form becomes the active window (gets focus).
4
Deactivate – Another form takes focus from this one.
5
Unload – User closes the form. Ask "Save before exit?" here.
6
Terminate – Form is completely removed from memory.
🧠 Where should you write "Are you sure you want to exit?" logic?
Initialize event
Load event
Unload event
The Unload event fires when the user tries to close the form — perfect for "Are you sure?" confirmation dialogs!
Show & Hide Methods
▼
VB6 ' Show a form Form2.Show ' Modeless (both usable) Form2.Show 1 ' Modal (must close Form2 first) ' Hide a form (still in memory) Form2.Hide ' Remove from memory completely Unload Form2
💡
Modal vs Modeless: Modal (Show 1) means the user MUST deal with that form before going back. Like a popup asking "Save file?". Modeless lets the user switch between forms freely.

Chapter 04
Variables, Data Types & Arrays
Variables & Data Types
▼
A variable is a named storage box in memory. You declare it with a name and a data type.
📦
Analogy: Variables are like labeled boxes. "Dim age As Integer" creates a box labeled "age" that can hold whole numbers only.
Integer
Whole numbers: -32768 to 32767
Long
Large whole numbers
Single/Double
Decimal numbers (float/double precision)
String
Text: "Hello", "BCA"
Boolean
True or False only
Date
Dates and times
Variant
Any type (avoid — uses more memory)
VB6 Dim studentName As String Dim marks As Integer Dim percentage As Single Dim isPassed As Boolean studentName = "Rahul" marks = 85 percentage = 85.5 isPassed = True
Scope of Variables
▼
Scope means "where can this variable be seen?" VB6 has three levels.
1
Local (Procedure Level) – Declared inside a Sub/Function. Only usable there. Dies when procedure ends.
Dim x As Integer inside a Sub
2
Module Level – Declared at the top of a form/module. All procedures in that form can use it.
Private score As Integer (top of form)
3
Global Level – Declared in a Module with Public. Every form and module can use it.
Public userName As String (in Module1)
🧠 Which keyword makes a variable accessible from ALL forms?
Dim
Private
Public
Public declared in a Module makes a variable available to ALL forms and modules in the project — it's truly global!
Arrays – Storing Multiple Values
▼
🗂️
Analogy: An array is like a row of numbered lockers. Instead of making 30 variables (student1, student2...), use one array: students(30).
VB6 ' Fixed-size array (5 elements: 0 to 4) Dim marks(4) As Integer marks(0) = 75 marks(1) = 88 ' Dynamic array (size decided at runtime) Dim names() As String ReDim names(10) ' resize ReDim Preserve names(20) ' resize, keep data ' 2D array (grid like roll x subject) Dim grid(5, 3) As Integer
⚠️
Preserve keyword: When you ReDim without Preserve, all existing data is LOST. Always use ReDim Preserve if you want to keep old data when resizing.

Chapter 05
Writing Code – Loops, Conditions & Decisions
If…Then…Else – The Decision Maker
▼
🚦
Think of it as a traffic signal: "IF marks ≥ 40, THEN you pass, ELSE you fail."
VB6 Dim marks As Integer marks = Val(txtMarks.Text) If marks >= 90 Then MsgBox "Grade: A" ElseIf marks >= 75 Then MsgBox "Grade: B" ElseIf marks >= 60 Then MsgBox "Grade: C" Else MsgBox "Grade: F – Better luck!" End If
🧠 Which keyword handles multiple conditions neatly without nested Ifs?
Do…Loop
Select Case
For…Next
Select Case is perfect when you're testing one variable against many values — much cleaner than chaining ElseIf!
Select Case
▼
Select Case is cleaner than multiple If-ElseIf when testing one variable.
VB6 Select Case day Case "Monday" MsgBox "Start of the week!" Case "Friday" MsgBox "Weekend coming!" Case "Saturday", "Sunday" MsgBox "It's the weekend!" Case Else MsgBox "Just another weekday" End Select
Loops: For…Next & While…Wend
▼
🔁
Analogy: A loop is like doing 10 push-ups. "For i = 1 to 10: do a push-up: Next i"
VB6 – For…Next ' Print numbers 1 to 5 Dim i As Integer For i = 1 To 5 Print i Next i ' Step backwards For i = 10 To 1 Step -1 Print i Next i
VB6 – While…Wend ' Keep going while condition is true Dim num As Integer num = 1 While num <= 5 Print num num = num + 1 Wend
⚠️
Infinite loop danger: If the condition in a While loop never becomes False, your program freezes! Always make sure the loop variable changes.

Multiple Document Interface (MDI)
What is MDI?
▼
🪟
Real-life example: Microsoft Word with multiple documents open — each document is a Child form, and the main Word window is the MDI Parent (container).
MDI Parent
The container window. Can hold many child forms. Add via Project → Add MDI Form.
Child Form
Set MDIChild = True on a regular form. It lives inside the parent.
ActiveForm
MDIForm1.ActiveForm gives you the currently focused child.
VB6 – Open new child Dim newDoc As New frmChild newDoc.Caption = "Document " & docCount newDoc.Show
🧠 What property must be set to True to make a form an MDI Child?
IsChild
MDIChild
ChildForm
Set MDIChild = True in the Properties window and the form automatically becomes a child that lives inside the MDI Parent form!
QueryUnload – Safe Closing
▼
QueryUnload fires before Unload — letting you CANCEL the close if needed.
VB6 – QueryUnload Private Sub Form_QueryUnload(Cancel As Integer, _ UnloadMode As Integer) Dim ans As Integer ans = MsgBox("Save before closing?", vbYesNoCancel) If ans = vbCancel Then Cancel = 1 ' Stop the close! End If End Sub
⚠️
Remember: Setting Cancel = 1 stops the form from closing. This is where you give users the "Save changes?" prompt before they lose their work.

Chapter 09
ActiveX Data Objects (ADO) & Databases
ADO Architecture: OLE DB & ADO
▼
1
OLE DB – Microsoft's low-level interface to access ALL types of data (databases, spreadsheets, text files). It's the bridge between your app and data.
2
ADO (ActiveX Data Objects) – A higher-level, easier wrapper over OLE DB. You use ADO in your VB code to read/write data without knowing OLE DB internals.
3
ODBC – Older standard. The ODBC Data Source Administrator lets you create named connections (DSN) to databases.
🔌
Think of it: Your VB app → ADO → OLE DB → Database. ADO is the translator that makes it easy to talk to the database.
Connecting to a Database
▼
VB6 – ADO Connection ' Step 1: Add reference to ADO ' Project → References → Microsoft ActiveX Data Objects Dim conn As New ADODB.Connection Dim rs As New ADODB.Recordset ' Step 2: Open connection conn.Open "Provider=Microsoft.Jet.OLEDB.4.0;" & _ "Data Source=C:\myDB.mdb" ' Step 3: Open recordset rs.Open "SELECT * FROM Students", conn ' Step 4: Read data Do While Not rs.EOF MsgBox rs("Name") rs.MoveNext Loop rs.Close conn.Close
⚠️
Always close connections! Leaving connections open wastes memory and can lock the database file. Always rs.Close and conn.Close when done.
Bound Controls & Cursors
▼
Bound Control
A control linked directly to a database field. Changes automatically sync.
ADO Data Control
Drag-drop control that manages DB connection. Other controls bind to it.
Cursor
A pointer that navigates through recordset rows. Types: Static, Forward-Only, Keyset, Dynamic
RecordSet
The set of rows returned by a SQL query. You navigate it row by row.
🧠 What does rs.EOF return True for?
When connection opens
When we've gone past the last record
When the record is empty
EOF (End Of File) becomes True after you've moved past the last record. This is why we check "Do While Not rs.EOF" in loops!

Crystal Reports & Data Reports
Crystal Reports Basics
▼
Crystal Reports is a tool that creates professional printable reports from database data — like report cards, invoices, and sales summaries.
1
Wizard Method – Use Crystal Reports Wizard. Choose database, table, fields. It auto-designs the layout for you.
2
Manual Method – Design the report yourself in the Design/Preview window. Drag fields, add text, format sections.
3
Sections – A report has: Report Header, Page Header, Detail (data rows), Page Footer, Report Footer.
4
Link to VB6 – Add CrystalReport control on a form. Set ReportFileName property. Call .Action = 1 to display.
Data Report Designer
▼
📊
VB6's built-in report tool: Data Report Designer is simpler than Crystal Reports and comes with VB6. Best for quick, simple reports.
Data Report
VB6's built-in report generator (simpler than Crystal)
Data Environment
Provides the database connection for the report
RptLabel
Report label control — like a Label but for reports
RptTextBox
Displays a database field value in the report
RptLine/Shape
Decorative lines and shapes for report layout
🧠 Which section of a report appears only once, at the very top?
Page Header
Report Header
Detail Section
Report Header appears once at the very beginning of the entire report — like a title page. Page Header appears at the top of EVERY page.
